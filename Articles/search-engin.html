What happened when you Press Search engine Like www.google.com

All it work using some processes that you will know in this Article

What they are and how they Work.

The Domain Name Server(DNS)

A DNS is a method of Finding IP Address for a web or application, When you Enter Your Client Softwares to Search Somthing like www.google.com or Wikipedia.org, it's DNS work to find what you want by transferring your request into the IP Address of the Website You are looking for in word wide web(www).

It's DNS work to recognize all the web or Application IP addresses that it will be difficult for Users to recognize, after recognizing the Name that assigned to the IP address.

EXAMPLE :
If 8.8.8.8 IP = AmirKano.tech,
When you type AmirKano or AmirKano.tech it's DNS work to take you to 8.8.8.8 IP Adress that represents AmirKano the Web you a Looking for, and all this happens Using UDP(User Datagrams Protocol) or TCP.

The TCP/IP.

A Transmission Control Protocol/Internet Protocol(TCP/IP), is a Communication suite protocol use to communicate internet and Devices.

The TCP works under some Layers protocol like, Data Link Layer, Internet Layer, Transport Layer and other more.

The Firewall.

A Firewall is a network security device that monitors and filters incoming and outgoing network traffic based on an organization's previously established security policies, It's between Users client and the internet.

A firewall is essentially the barrier that sits between a private internal network and the public Internet works for the Safety of Users and the network provider monitoring system.

The HTTPS/SSH.

A Hyper Text Transfer Protocol Secure(HTTPS), is a secured method by an SSL certificate, that show the Security of a web. When a website have HTTPS it means that the web is secured and more safe for User, and a lock key appear in the the top search of the web.

Different Between HTTP and HTTPS.

A HTTPS is Encrypted the data accept from User using SSL or TLS protocol and the web code for security while the HTTP accept it without Encrypted.

The SSH is also a part of HTTPS protocol that performs the encryption of the data.

The Loading Balancer

A Loading Balancer is a software and hardware that stand between Users and Servers. A Loading Balancer accept all Users request in One time on a web or software and send it to to several servers that a sign to.

It's Loading Balancer work to allow all users get what they requested from server in from different locations and different requests all in one time.

A Web Server

A Web Server is a Softwares work between Local clients to internet to Router to loading Balancer and then Server connect using HTTPS or HTTP protocol.

On the hardware , A web server is a computer that stores web server software and a website's component files like HTML documents, images, CSS stylesheets, and other more.

A web server connects to the Internet and supports physical data interchange with other devices connected to the web.

The Application Server

An application server is a modern form of platform middleware. It is system software that resides between the operating system (OS) on one side, the external resources (such as a database management system (DBMS), communications and Internet services) on another side and the users' applications on the third side.

The Databases(DB)

A database is an organized collection of structured information, or data, typically stored electronically in a computer system.

A Databases often store information about people, such as customers or users. For example, social media platforms use databases to store user information, such as names, email addresses and user behavior. The data is used to recommend content to users and improve the user experience.

All these methods are the methods Softwares and Website we use day to day work on, it's with those methods all they are operating.
